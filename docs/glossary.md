# Glossary

Terms that show up in MuOxi docs and code, defined precisely. Sorted
alphabetically.

**Account**
A persisted login identity. One name, one argon2id password hash, zero
or more `Character`s. Stored in the `accounts` table. See
[`db/src/structures.rs`](../db/src/structures.rs).

**`at_*` hooks**
Lifecycle event listener methods on the `Hook` trait — `at_login`,
`at_disconnect`, `at_object_created`, `at_pre_destroy`, `at_pre_move`,
`at_post_move`, `at_say`. Downstream MUDs implement them to react to
engine events. See [extension-guide.md § Hooks](extension-guide.md#hooks).

**`AuthBuffer`**
Per-session scratch space (`pending_name`, `first_password_attempt`) used
by the state machine to carry partial credentials across state
transitions. Cleared on successful auth, failed auth, or session loss.

**Attribute** (`ObjectAttribute`)
A `(key → JSON value)` entry attached to an `Object`. The freeform
state bag. Adding a new attribute doesn't require a schema migration.

**Cache** (`db::cache`)
A factory for Redis connections. Reads `REDIS_SERVER` from env.

**`Cachable`**
The trait that types implement to round-trip themselves through Redis
under keys of the form `<TypeName>:<UID>:<field>`. Currently the only
implementer is `CacheSocket`.

**`CacheSocket`**
A `Cachable` representing the per-connection socket state (IP, port,
UID). Persisted in Redis at connect, deleted at disconnect. Survives
brief Redis outages but not a full restart.

**Character**
An `Object` with `type_key = "character"`, linked to an `Account` via
the `character_accounts` table. One account can have many characters.
The in-world avatar a player puppets.

**`CharacterAccount`**
A link-table row tying a character `Object` to an `Account` with a
0-indexed `ordinal` (the slot in the player's character list).

**`Client`**
The per-connection state struct. Holds the `Framed<TcpStream, LinesCodec>`,
the connection state (`ConnStates`), the session UID, the optional
`account_uid` / `character_uid`, and the `AuthBuffer`. Lives for the
duration of one TCP connection.

**`Command`**
The trait implemented by anything a player can type. Registered with
`Registry::register_command(Arc::new(MyCmd))`. Receives a
`CommandContext` at invocation time.

**`CommandContext`**
What a `Command::execute_cmd` gets: `&mut Client`, `Arc<Registry>`,
`Arc<WorldApi>`, and the raw `args: &str`.

**`Conn`**
Type alias resolved by the active Cargo feature to either
`SqliteConnection` or `PgConnection`. Diesel 2.x query helpers all need
`&mut Conn`.

**`ConnStates`**
The 8-state enum describing where a connection is in its lifecycle —
`AwaitingName`, `AwaitingPassword`, `AwaitingNewName`,
`AwaitingNewPassword`, `ConfirmNewPassword`, `MainMenu`, `Playing`,
`Quit`.

**`DatabaseHandler`**
The DB facade owned by `WorldApi`. Carries the `Conn` plus all the
typed repos (`accounts`, `objects`, `attributes`, `tags`,
`character_accounts`). Construct via `DatabaseHandler::connect()`,
which runs embedded migrations.

**`DEV_AUTOLOGIN`**
Env var that, when set, makes the server skip the auth state machine
and drop new connections straight into `Playing` as a throwaway "Dev"
character. Developer convenience; not for production.

**Exit**
An `Object` with `type_key = "exit"`. Convention: lives inside the
source room (its `location_uid`), carries a `destination` attribute
pointing at the target room's UID.

**Framework-level vs downstream-level**
A distinction this codebase uses repeatedly. "Framework" = MuOxi the
engine. "Downstream" = a Rust developer using MuOxi to build their MUD.
Some APIs are framework-internal; others are public extension points.

**`Hook`**
The trait downstream MUDs implement to listen for lifecycle events.
Registered via `Registry::register_hook`. As of v0.1 only `at_login`
and `at_disconnect` are fired by the engine; the rest are declared but
unwired.

**`HookContext`**
Read-mostly context handed to hook implementations: `&WorldApi`,
`Option<UID>` session.

**Item**
An `Object` with `type_key = "item"`. Pickable, droppable, can live
inside rooms or other containers.

**Lock**
A string returned by `Command::lock()` evaluated against the actor's
permission tags. Today's tiny DSL supports `all()`, `false`, and
`perm(NAME)`. See [`muoxi/src/server/locks.rs`](../muoxi/src/server/locks.rs).

**Mob**
An `Object` with `type_key = "mob"`. An NPC. Persistence-identical to a
character but NOT linked to an account.

**`Object`**
The universal in-world entity row: `(uid, type_key, name, location_uid?,
created_at)`. The `type_key` discriminates kinds (room / item / mob /
character / exit / your-custom-type).

**Object UID**
A `BIGINT` generated by `gen_uid()` = `(unix_secs << 32) | rand32`.
Schema enforces `uid > 0`.

**`PROXY_ADDR`**
The TCP bind address for `muoxi_server`. Default `127.0.0.1:8000`.
Override in docker compose to `0.0.0.0:8000` for external access.

**Registry**
The central runtime index of TypeClasses, Commands, and Hooks plus a
shared `Arc<WorldApi>`. Built once in `main()`, threaded into every
spawned `process()` task. The thing downstream MUDs register against.

**Room**
An `Object` with `type_key = "room"`. Typically has `location_uid =
None` (rooms aren't usually contained by other objects). Holds
characters, items, mobs, and exit objects.

**`seed_world`**
The idempotent first-boot world seeder in
[`muoxi/src/server/seed.rs`](../muoxi/src/server/seed.rs). Ensures a
starting room (tagged `(starting-room, system)`) exists.

**Session UID**
A per-connection random ID generated at accept and persisted in Redis.
Distinct from the character UID. The session UID is socket-tied; the
character UID is in-world-tied. Plan 6 separated these two concepts;
pre-Plan-6 code conflated them.

**`SessionConfig`**
A small struct passed into each `process()` task, currently carrying
just `dev_autologin_room`. Will grow as more per-session policies are
added.

**Tag** (`ObjectTag`)
A `(key, category)` label attached to an `Object`. Used for cross-object
lookups (`objects_with_tag`) and as the storage backing for the
`perm(NAME)` lock primitive (`(NAME, "permission")`).

**TypeClass**
The trait describing an in-world entity *type* (as opposed to an
instance). Pairs a `type_key` with default attributes, default tags,
default command set, and a lock map. As of v0.1 only the defaults for
the `"character"` type are auto-applied at creation; generic
auto-application is on the v0.2 roadmap.

**`WEB_ADDR`**
The bind address for `muoxi_web`. Default `127.0.0.1:8080`.

**`WorldApi`**
The DB facade command handlers and hooks use to read/write the world.
Wraps `DatabaseHandler` in `Arc<Mutex<>>`. The only place gameplay code
should touch Diesel.
