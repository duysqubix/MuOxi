# db crate (library)

Shared persistence + caching layer. Used by `muoxi`, `tester`, `benchmarks`. Backend-agnostic: defaults to SQLite via `db-sqlite`; Postgres available via `--no-default-features --features db-postgres`. Wraps Diesel 2.x for durable state, Redis 0.27 for transient/cache state, and `serde_json` for import/export.

## STRUCTURE

```
db/
├── Cargo.toml             # library; [features] db-sqlite (default) / db-postgres
└── src/
    ├── lib.rs             # DatabaseHandler facade; re-exports diesel
    ├── conn.rs            # Conn type alias + establish() + WAL pragmas
    ├── utils.rs           # UID type, gen_uid() (rand 0.8), JSON helpers
    ├── schema.rs          # portable schema (BigInt/Text); accounts + objects/
    │                      # attributes/tags/character_accounts
    ├── structures.rs      # Account Diesel model + DatabaseHandlerExt trait
    ├── objects/
    │   ├── mod.rs                    # module root + re-exports
    │   ├── object.rs                 # Object + ObjectRepo (generic entity)
    │   ├── attribute.rs              # ObjectAttribute + AttributeRepo (JSON bag)
    │   ├── tag.rs                    # ObjectTag + TagRepo (labels + categories)
    │   └── character_account.rs      # CharacterAccount + repo (acct ↔ char link)
    ├── cache.rs           # Redis Connection factory
    └── cache_structures/
        ├── mod.rs         # Cachable trait + key-naming convention
        └── socket.rs      # CacheSocket: per-client Redis-backed state
```

## BACKENDS

| Feature | Backend | When |
|---|---|---|
| `db-sqlite` (default) | SQLite via bundled `libsqlite3-sys` | local dev, hobby deployments, CI |
| `db-postgres` | Postgres via `libpq-dev` | production, multiple writers, replicas |

Build:
```bash
cargo build                                            # SQLite (default)
cargo build --no-default-features --features db-postgres   # Postgres
```

Exactly one backend must be active — a `compile_error!` in `conn.rs` enforces this.

## CORE TYPES

| Type | Purpose |
|------|---------|
| `Conn` | Type alias resolved by the active feature: `SqliteConnection` or `PgConnection`. |
| `UID = i64` | 64-bit unique id. Generated by `gen_uid()` = `(unix_secs << 32) \| rand32`. Schema constraint `uid > 0`. |
| `DatabaseHandler` | Owns one `Conn` + handlers (`accounts`, `objects`, `attributes`, `tags`, `character_accounts`). Construct via `DatabaseHandler::connect()`. |
| `DatabaseHandlerExt<T>` trait | CRUD contract for typed handlers: `upsert`, `insert`, `remove`, `get`, `get_batch`, `get_range`, `exists`. All methods take `&mut Conn`. |
| `account::Account` | Login identity: `uid, name, password_hash, email, created_at`. |
| `objects::Object` | Generic in-world entity: `uid, type_key, name, location_uid?, created_at`. Self-referential FK for containment. |
| `objects::ObjectRepo` | CRUD on objects: `create`, `get`, `delete`, `move_to`, `rename`, `list_by_type`, `contents_of`. |
| `objects::ObjectAttribute` / `AttributeRepo` | Per-object key→JSON bag. `set` upserts via ON CONFLICT, `get`/`all` parse JSON, `delete` removes. |
| `objects::ObjectTag` / `TagRepo` | Per-object (key, category) labels. Idempotent `add`, cross-object `objects_with` lookup. |
| `objects::CharacterAccount` / `CharacterAccountRepo` | Character ↔ account link with ordinal. Replaces the old `account_characters` join. |
| `Cache` | Redis connection factory. `Cache::new()` reads `REDIS_SERVER`, defaults `redis://127.0.0.1`. |
| `Cachable` trait | Generic redis serialization via key format `Type:UID:fieldName`. |
| `CacheSocket` | Concrete `Cachable` impl storing `(ip, port, uid)` per connected socket. Uses redis `mset`. |

## CONVENTIONS

- Diesel 2.x style: NO `#[macro_use] extern crate diesel;`. Macros are namespaced (`diesel::table!`, `diesel::allow_tables_to_appear_in_same_query!`).
- All ORM record structs:
  - derive `Queryable, Insertable, AsChangeset, Clone, Serialize, Deserialize`
  - declare `#[diesel(table_name = <table>)]` on the struct
- All `DatabaseHandlerExt` methods take `conn: &mut Conn`.
- Cache key naming convention is FIXED: `<TypeName>:<UID>:<field>`.
- `CacheSocket::name` is hard-coded to `"Socket"`.
- `db` re-exports `diesel` for downstream crates: `use db::diesel::prelude::*;` works without adding a direct `diesel` dep.

## DATABASE SCHEMA

Built up by two migrations under `migrations/`:

```sql
-- 2026-05-07-000000_initial
accounts(uid PK, name unique, password_hash, email, created_at)

-- 2026-05-07-000100_objects (drops the original characters + account_characters tables)
objects(uid PK, type_key, name, location_uid? -> objects(uid) SET NULL, created_at)
object_attributes(object_uid -> objects, key, value,
                  PK(object_uid, key), CASCADE on object delete)
object_tags(object_uid -> objects, key, category,
            PK(object_uid, key, category), CASCADE on object delete)
character_accounts(object_uid PK -> objects CASCADE,
                   account_uid -> accounts CASCADE, ordinal)
```

`uid > 0` CHECK constraint on every UID column. Portable across SQLite and Postgres.

## ADDING A NEW IN-WORLD TYPE

You do NOT change the schema. Register a new `type_key` value (e.g. `"weapon"`,
`"npc"`, `"chest"`) and use `ObjectRepo::create(conn, "weapon", ...)`. Per-type
gameplay state goes into `ObjectAttribute` (e.g. `attributes.set(uid, "damage", json!(7))`).
Plan 4 introduces a `TypeClass` registry that pairs `type_key` with command sets
and lifecycle hooks; until then, downstream code just calls into the repos
directly.

## ANTI-PATTERNS

- DO NOT use `BIGINT[]`, `JSONB`, `LISTEN/NOTIFY`, `to_tsvector`, or other Postgres-only types in the core schema. Keep it portable.
- DO NOT use `redis::Commands::set_multiple` - deprecated in 0.27, use `mset`.
- DO NOT call `DatabaseHandler::connect()` from inside an async task without offloading - Diesel 2.x is sync-blocking.
- DO NOT pass `&Conn` (immutable) to any query helper. All Diesel 2.x query execution requires `&mut Conn`.
- DO NOT bring back JSON-canonical/watchdog. The DB is the single source of truth; `json/` is import/export only.
- DO NOT bypass the repos. Engine code calls `db.objects.create(...)`, never `diesel::insert_into(objects::table)` directly.
- DO NOT add a separate `characters` table back. Characters are objects with `type_key = "character"`; their account link lives in `character_accounts`.

## ADDING A NEW TABLE

1. Write `migrations/<date>_<name>/up.sql` and `down.sql`. Use portable types only.
2. Add a `diesel::table!` block to `db/src/schema.rs`.
3. New module defining record + handler in `db/src/structures.rs` or under `db/src/`.
4. Derive `Queryable, Insertable, AsChangeset, Clone, Serialize, Deserialize` and add `#[diesel(table_name = <table>)]`.
5. `impl DatabaseHandlerExt<YourRecord> for YourHandler` (`&mut Conn`).
6. Add field to `DatabaseHandler` struct + initialize in `connect()`.

## SYSTEM REQUIREMENT

Default `db-sqlite` build: **none**. SQLite is bundled.
Postgres `db-postgres` build: `libpq-dev` (Debian/Ubuntu) or equivalent.
