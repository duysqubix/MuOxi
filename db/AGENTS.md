# db crate (library)

Shared persistence + caching layer. Used by `muoxi`, `tester`, `benchmarks`. Wraps Diesel 2.x + Postgres for durable state, Redis 0.27 for transient/cache state, and `serde_json` for flat-file canonical sources.

## STRUCTURE

```
db/
├── Cargo.toml             # library only ([lib] name="db", path="src/lib.rs")
└── src/
    ├── lib.rs             # DatabaseHandler facade (no `extern crate diesel` - 2.x style)
    ├── utils.rs           # UID type, gen_uid() (rand 0.8 range syntax), JSON helpers
    ├── schema.rs          # Diesel-generated; uses `diesel::table!` (namespaced)
    ├── structures.rs      # Diesel models: Account, Character + DatabaseHandlerExt trait
    ├── cache.rs           # Redis Connection factory
    └── cache_structures/
        ├── mod.rs         # Cachable trait + key-naming convention
        └── socket.rs      # CacheSocket: per-client Redis-backed state
```

> Note: `clients.rs` was deleted during modernization. It was dead code (no migration, never re-exported from `lib.rs`).

## CORE TYPES

| Type | Purpose |
|------|---------|
| `UID = i64` | 64-bit unique id. Generated by `gen_uid()` = `(unix_secs << 32) \| rand32`. Schema constraint `uid > 0`. |
| `DatabaseHandler` | Owns one `PgConnection` + sub-handlers (`accounts: AccountHandler`, `characters: CharacterHandler`). Construct via `DatabaseHandler::connect()` (panics on failure). |
| `DatabaseHandlerExt<T>` trait | CRUD contract: `upsert`, `insert`, `remove`, `get`, `get_batch`, `get_range`, `exists`. **All methods take `&mut PgConnection`** (Diesel 2.x). Implement for any new ORM record. |
| `RecordVector<T>` / `RecordHashMap<T>` | Newtype wrappers; `From<RecordHashMap<T>> for RecordVector<T>` exists for JSON-import paths. |
| `Cache` | Redis connection factory. `Cache::new()` reads `REDIS_SERVER`, defaults `redis://127.0.0.1`. |
| `Cachable` trait | Generic redis serialization via key format `Type:UID:fieldName`. Methods: `dump`, `load`, `destruct`, `make_key`, `create_tag`. |
| `CacheSocket` | Concrete `Cachable` impl. Stores `(ip, port, uid)` per connected socket. Uses redis `mset` (replacing the deprecated `set_multiple`). |

## CONVENTIONS

- Diesel 2.x style: NO `#[macro_use] extern crate diesel;`. Macros are namespaced (`diesel::table!`, `diesel::allow_tables_to_appear_in_same_query!`).
- All ORM record structs:
  - derive `Queryable, Insertable, AsChangeset, Clone, Serialize, Deserialize`
  - declare `#[diesel(table_name = <table>)]` on the struct (Diesel 2.x attribute syntax)
- All `DatabaseHandlerExt` methods take `conn: &mut PgConnection`. Callers must pass `&mut db.handle`. The owning `DatabaseHandler` value must therefore be `let mut db = DatabaseHandler::connect();`.
- Cache key naming convention is FIXED: `<TypeName>:<UID>:<field>`. Each field is its own redis key (NOT a hmap) so per-field TTLs are possible. Documented in `cache_structures/mod.rs`.
- `CacheSocket::name` is hard-coded to `"Socket"` - matches the prefix you'll see in `redis-cli KEYS '*'`.

## DATABASE SCHEMA

Defined by `migrations/2020-01-21-170702_muoxi/up.sql`:

```sql
Accounts(uid BIGINT PK, name varchar(64), password varchar(64),
         email varchar(64), characters BIGINT[] NULL)
Characters(uid BIGINT PK, account BIGINT, name varchar(64))
```

Both tables enforce `uid > 0` via CHECK constraint. `Account.characters` mirrors per-account character UID list - kept in sync manually (no FK / cascade).

`schema.rs` is generated by Diesel CLI. Do not modify directly - edit migrations and re-run `diesel migration run`.

## ANTI-PATTERNS

- DO NOT edit `db/src/schema.rs` by hand. It is generated by Diesel (config: `diesel.toml` -> `db/src/schema.rs`). Add a new migration and re-run.
- DO NOT use `redis::Commands::set_multiple` - deprecated in 0.27, use `mset`. Already migrated in `cache_structures/socket.rs::Cachable::dump`.
- DO NOT call `DatabaseHandler::connect()` from inside an async task without offloading - Diesel 2.x is still sync-blocking (we did NOT migrate to async Diesel). Existing usage in `watchdog.rs` works because the watchdog itself is sync (`fn main`, no `#[tokio::main]`).
- DO NOT mix Redis HMAP (`hset`) with the existing `set`-per-field convention. `Cachable::dump` uses `mset` of individual key/value pairs.
- DO NOT pass `&PgConnection` to any query helper. All Diesel 2.x query execution requires `&mut PgConnection`.

## ADDING A NEW TABLE

1. `diesel migration generate <name>` → write `up.sql` + `down.sql`
2. `diesel migration run` → updates `db/src/schema.rs` AUTOMATICALLY
3. New module in `db/src/structures.rs` (or sibling) defining record + handler
4. Derive `Queryable, Insertable, AsChangeset, Clone, Serialize, Deserialize` and add `#[diesel(table_name = <table>)]`
5. `impl DatabaseHandlerExt<YourRecord> for YourHandler` (remember: `&mut PgConnection`)
6. Add field to `DatabaseHandler` struct + initialize in `connect()`

## SYSTEM REQUIREMENT

`libpq-dev` (Debian/Ubuntu) or equivalent must be installed for any binary that links this crate. `cargo check` works without it; `cargo build` does not.
